Cookie是什麼

最後修改：

簡介

因HTTP協定無狀態而生

因HTTP協定為無狀態(stateless)之協定，基本上Web Server對client之請求不會留下狀態記錄，試想Web Server端之應用程式面對同時向它作請求之許多clients，如何分辨此次請求是來自那個client。針對此問題，HTTP協定中制定Cookie之機制


紀錄資訊

基本上cookie是用來記錄Web Server端應用程式之一些資訊，此資訊存放在瀏覽器端之電腦
Web Server端之應用程式可利用 Cookie 之機制儲存一些資訊在使用者瀏覽器中(利用Set-Cookie:)
這將使瀏覽器能記下一些資訊，以便往後再次請求相同之Web Server端應用程式時， 可附帶此資訊於請求訊息中(利用Cookie:)，Web Server 端應用程式就可取得原先存放在該使用者瀏覽器中之一些資訊
且所有的cookie都有時效限制，時間到了就自動消失


Server端的應用

Cookies要如何應用全由Web Server端之應用程式設計者決定，例如，Web Server端之應用程式或程式開發平台常利用cookie記錄session id，再利用session id 追蹤某使用者以及該使用者在執行某應用程式時所需之一些變數，這些變數即一般所謂之session variables


安全性疑慮

有些應用程式利用cookie記錄一些較有安全疑慮之資訊，如帳號密碼，因此有人擔心使用cookie會有安全疑慮


Cookie的設定

由Web Server端之應用程式設定cookie, Web Server 以Set-Cookie: 設定cookie於HTTP回應訊息中

Set-Cookie: NAME=VALUE; expires=DATE; path=PATH; domain=DOMAIN_NAME;
  • NAME=VALUE：定義cookies
  • expires=DATE：定義 cookies 之有效期
  • domain與path：定義 cookies 之有效範圍(那個server之那個應用)

若已存在Cookie，會自動回報Server

瀏覽器在向 Web server 作請求時, 若有合於此server之有效cookie時會以Cookie: 回報cookie

Cookie: NAME=VALUE;
